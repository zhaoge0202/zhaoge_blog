# 回溯模板怎么套？

> 回溯就是「深度优先 + 撤销选择」：一路往下试，走不通就退回来，把现场还原，再换下一条路。

## 为什么叫回溯，而不是普通 DFS？

DFS 关心的是「能不能走到某个点」；回溯关心的是「所有合法路径长什么样」。两者走法一样，都是递归往下钻，差别在于：

- 回溯会显式维护一条**当前路径**（path）；
- 试完一个分支后，必须**撤销**刚才的选择，把 path 还原，才能把同一条 path 复用给兄弟分支。

可以把它想成在一棵「选择树」上走路：每一层对应一次决策，根是「还什么都没选」，叶子是「凑出了一份完整方案」。路径上走过的节点拼起来，就是答案的一部分。

## 三要素：路径、选择、结束

动笔前先把这三件事说清楚，参数才不会乱：

| 要素      | 含义           | 代码里常见形态                      |
| --------- | -------------- | ----------------------------------- |
| 路径 path | 已经选了什么   | `List` / 数组                       |
| 选择列表  | 当前还能选什么 | 下标区间、`used[]`、剩余候选        |
| 结束条件  | 什么时候收答案 | path 长度够了、目标值耗尽、棋盘放满 |

再补一条：**撤销**。path 通常是复用对象，当前分支试完后必须 `remove` 或改回标记，否则下一分支会带着脏状态继续走。

## 通用框架

伪代码几乎可以原样套到组合、排列、子集、棋盘题上：

```text
void backtrack(path, 选择上下文):
    if 满足结束条件:
        记录 path 的一份拷贝
        return

    for choice in 当前可选:
        if 不可行:          // 可行性剪枝
            continue
        做选择：path 加入 choice，更新标记
        backtrack(path, 更新后的上下文)
        撤销选择：path 弹出，标记还原
```

写代码时有三个容易踩的点：

1. **收答案要拷贝**。`ans.add(path)` 会把引用塞进去，后面 path 一变，答案全跟着变；应 `ans.add(new ArrayList<>(path))`。
2. **组合 / 子集用 start 下标**，保证只往右选，天然去重「顺序不同但集合相同」的重复。
3. **排列用 used[]**，同一元素在一条路径里只能出现一次；每层都可以从 0 扫到末尾。

## 剪枝：三种常见类型

剪枝不是加分项，是能不能过的关键。按动机分三类：

**可行性剪枝**——这个选择本身就不可能合法。  
例如组合总和里 `target - nums[i] < 0` 直接跳过；N 皇后里同行 / 同列 / 同对角线已有皇后就别放。

**最优性剪枝**——已经比已知最优差了，不必继续。  
求「最少步数 / 最少数量」时，若当前 path 长度已经 ≥ 已找到的最优解，直接 return。注意：只在「求最优」时有用，求「所有方案」时没有「最优」可剪。

**去重剪枝**——输入有重复元素时，避免生成相同答案。  
套路是：**先排序**，再在同一层跳过「与前一个相同」的选择。同一层重复会产生重复答案；同一路径上按顺序使用相同值，往往是合法的。

组合 / 子集去重（用 start）：

```text
if i > start and nums[i] == nums[i - 1]:
    continue
```

全排列去重（配合 used）：

```text
if i > 0 and nums[i] == nums[i - 1] and not used[i - 1]:
    continue
```

后一句的直觉：同层里，相同数字只允许「按出现顺序」被选一次，避免 `[1a, 1b, 2]` 和 `[1b, 1a, 2]` 这种仅下标不同的重复排列。

## 代表题 1：子集（含重复元素去重）

题意：给定可能含重复数字的数组，返回所有不重复子集。空集也算。

思路：每个节点都是一个合法子集（包括根上的空集），所以**一进函数就先收一份 path**，再从 `start` 往后枚举「选谁」。

```text
function subsetsWithDup(nums):
    sort(nums)
    ans = []
    path = []

    function dfs(start):
        ans.add(copy(path))          // 每个节点都是答案
        for i = start .. n-1:
            if i > start and nums[i] == nums[i-1]:
                continue             // 同层去重
            path.add(nums[i])
            dfs(i + 1)               // 只能往后选
            path.pop()

    dfs(0)
    return ans
```

关键细节：

- `i + 1` 保证每个元素在一条路径里最多用一次，也保证子集内部有序。
- 去重条件是 `i > start`，不是 `i > 0`。`i > 0` 会误伤「跨层使用相同值」的合法情况，例如 `[1, 2, 2]` 里的 `[2, 2]`。
- 输入为空时，dfs(0) 仍会收一次空 path，答案是 `[[]]`，符合定义。

组合题几乎同一套骨架，只是**只在 `path.size() == k` 时收答案**，并且可以把上界剪成 `i <= n - (k - path.size()) + 1`：剩余数字已经凑不够 k 个就别循环了。

## 代表题 2：组合总和

题意：候选数组 `candidates`（元素可重复使用），找出所有和为 `target` 的组合。组合内顺序不敏感。

```text
function combinationSum(candidates, target):
    sort(candidates)                 // 方便剪枝
    ans = []
    path = []

    function dfs(start, remain):
        if remain == 0:
            ans.add(copy(path))
            return
        for i = start .. n-1:
            if candidates[i] > remain:
                break                // 后面更大，直接停
            path.add(candidates[i])
            dfs(i, remain - candidates[i])   // 仍从 i 开始：可重复用
            path.pop()

    dfs(0, target)
    return ans
```

和子集题对比，有两处容易混：

| 点       | 子集 / 组合             | 组合总和            |
| -------- | ----------------------- | ------------------- |
| 递归下标 | `i + 1`（每个数用一次） | `i`（当前数可再用） |
| 结束条件 | 长度 / 每个节点         | remain == 0         |
| 剪枝     | 剩余个数不够            | 当前值 > remain     |

若题目改成「每个数最多用一次」，递归改成 `dfs(i + 1, ...)`，再叠加上面的同层去重即可。

N 皇后是同一框架的棋盘版：path 记录每行放在哪一列，选择列表是尚未占用的列与对角线，可行性剪枝用三个布尔数组（列、主对角、副对角）判断。

## 复杂度怎么估

回溯的时间复杂度几乎总是「选择树有多大」，而不是表面上的 O(n)。

- **分支因子 × 深度**：全排列大约 n! 个叶子；子集 2^n 个节点；N 皇后上界接近 n!，剪枝后实际远小。
- **剪枝只改常数与平均规模**，最坏上界往往不变。面试里可以说：「最坏仍是指数级，剪枝把无效分支砍掉，实际可解规模取决于数据」。
- **空间**：递归深度最多 O(n)（或棋盘边长），加上 path 的 O(n)；答案列表本身的空间通常不计入算法额外空间，但生成全部方案时输出规模可能很大。

不要随手写 O(n) 或 O(n log n)——除非你先排序再搜索，排序那一段才是 n log n，搜索主体仍是指数。

## 边界与 start 下标

写完模板后，建议用这几组边界自测：

| 场景                       | 期望行为                                             |
| -------------------------- | ---------------------------------------------------- |
| 空数组                     | 子集返回 `[[]]`；排列返回 `[[]]`；组合 k>0 返回 `[]` |
| 全是重复元素，如 `[2,2,2]` | 去重后子集只有 `[], [2], [2,2], [2,2,2]`             |
| target = 0                 | 组合总和直接收一份空组合，或按题意返回 `[[]]`        |
| k = 0                      | 组合题通常返回 `[[]]`                                |
| 候选有 0 / 负数            | 组合总和要额外处理，否则可能死循环                   |

`start` 下标的本质：**固定「只往右看」的扫描方向**，用顺序消掉排列意义上的重复。排列题没有 start，因为顺序本身就是答案的一部分，改用 `used[]` 控制「本路径是否已用」。

## 和 DP 怎么取舍

同一道题有时既能回溯也能 DP，看目标：

| 目标                       | 更合适的方法                            |
| -------------------------- | --------------------------------------- |
| 列出所有方案               | 回溯（天然产出 path）                   |
| 方案个数 / 最优值          | DP 或记忆化搜索                         |
| 既要最优值又要任意一组方案 | 先 DP 求值，再反推 path；或带剪枝的搜索 |

经验法则：

- 状态好定义、有重叠子问题 → DP 更快。
- 约束复杂、方案结构本身就是答案 → 回溯更直接。
- 「组合总和 IV」求排列个数用 DP；「组合总和」求组合列表用回溯——差一个「是否关心顺序 / 是否只要计数」。

## 小结

1. 回溯 = DFS + 撤销；写之前先钉死 path、选择列表、结束条件。
2. 组合 / 子集靠 `start` 只往右选；排列靠 `used[]`；收答案务必拷贝 path。
3. 剪枝分可行性、最优性、去重三类；有重复元素先排序再同层跳过。
4. 时间看选择树规模（常指数），空间看递归深度与 path；剪枝改实际耗时不改最坏量级表述。
5. 要方案用回溯，要计数或最优值优先 DP；两者边界用「输出形态」判断。
