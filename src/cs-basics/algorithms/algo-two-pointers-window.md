# 双指针和滑动窗口怎么用？

> 双指针把 O(n²) 扫组合降到 O(n) 量级，前提是序列有序或可维护窗口性质。

先做一个判断：题目关心 **两个位置之间的关系**，先想双指针；关心 **连续子数组/子串** 且区间内要维护状态，先想滑动窗口。两者写法很像，语义不一样——窗口强调「一段连续区间 + 区间状态」。

## 三类指针形态

| 形态             | 指针怎么动                          | 典型题                                   |
| ---------------- | ----------------------------------- | ---------------------------------------- |
| **对撞（左右）** | 两端往中间收                        | 有序两数之和、接雨水变体、回文判断       |
| **同向 / 快慢**  | 同向推进，速度或职责不同            | 链表环/中点、原地去重、读写分离          |
| **滑动窗口**     | `right` 扩、`left` 缩，维护区间状态 | 最长无重复子串、最小覆盖子串、固定长窗口 |

读写指针可以看成同向指针的特例：`read` 扫全表，`write` 只写要保留的。链表上的快慢指针一笔带过——环检测、找中点、倒数第 k 个节点，细节放在 [链表与树套路](/cs-basics/algorithms/algo-linkedlist-tree-patterns.html) 里展开。

## 通用窗口模板

可变窗口的固定节奏：

```text
left = 0
for right in 0 .. n-1:
    把 s[right] 加入窗口，更新状态
    while 窗口不合法 / 已经可以收缩:
        更新答案（若是「最短」类，常在这里）
        移出 s[left]，left++
    更新答案（若是「最长」类，常在这里）
```

两处答案更新别混：

| 目标                 | 时机                                     |
| -------------------- | ---------------------------------------- |
| **最长**合法窗口     | 先 `while` 修到合法，再记 `right-left+1` |
| **最短**满足条件窗口 | 一旦满足就记答案，再继续缩 `left` 试更短 |

固定长度窗口更简单：先堆满 `k`，之后右侧进一个、左侧出一个，滚动统计即可；若还要窗口最值，往往要配单调队列。

状态常见维护物：元素和、字符计数、种类数、匹配了多少种目标字符。**扩缩时状态必须同步**——只动指针不改计数，是最高发的 bug。

## 代表题 1：有序两数之和 → 三数之和

有序数组找 `a[i] + a[j] == target`（`i < j`）：

```text
left, right = 0, n-1
while left < right:
    s = a[left] + a[right]
    if s == target: 找到
    else if s < target: left++   // 和偏小，左端必须变大
    else: right--                // 和偏大，右端必须变小
```

为什么敢只往一个方向走？因为有序性给了 **单调排除**：和太小时，右指针再往左只会更小，只能动左；和太大时同理。每一轮排除一个候选端点，所以是 **`O(n)`**，不是两层枚举。

数组无序时：要么先排序再对撞（总时间 **`O(n log n)`**），要么哈希一遍（时间 `O(n)`、额外空间 `O(n)`）。

**三数之和**是同一套路的嵌套：

```text
排序 a
for i = 0 .. n-1:
    若 a[i] 与 a[i-1] 相同: continue          // 固定端去重
    L, R = i+1, n-1
    while L < R:
        s = a[i] + a[L] + a[R]
        if s == 0:
            记录三元组
            L++, R-- 并跳过与当前相同的值     // 对内去重
        else if s < 0: L++
        else: R--
```

时间 **`O(n²)`**（外层 n、内层对撞线性），空间除排序外常数级。去重分两处：固定的 `i`，以及命中后的 `L/R`——漏一处就会重复三元组。

## 代表题 2：最长无重复子串

求字符串中无重复字符的最长子串长度。窗口语义：`[left, right]` 始终是「当前无重复」的一段。

```text
count = 空映射   // 字符 → 窗口内出现次数
left = 0, ans = 0
for right = 0 .. n-1:
    c = s[right]
    count[c] += 1
    while count[c] > 1:          // 出现重复，收缩到合法
        count[s[left]] -= 1
        left += 1
    ans = max(ans, right - left + 1)
return ans
```

走一遍 `abba`：

| 加入 | 窗口         | 动作                  | 最长 |
| ---- | ------------ | --------------------- | ---- |
| `a`  | `a`          | 合法                  | 1    |
| `b`  | `ab`         | 合法                  | 2    |
| `b`  | 最终窗口 `b` | 重复则 while 连续收缩 | 2    |
| `a`  | `ba`         | 合法                  | 2    |

注意收缩必须用 **`while`**，不能 `if` 只挪一步——`abba`、`aaabc` 这类输入会挂。

若改做「最小覆盖子串」：维护 `need`（目标频次）、`window`（当前频次）、`valid`（已满足的字符种类）。`valid == need 种类数` 时先记最短长度，再缩左边界；`valid--` 要发生在减少计数 **之前**，否则临界状态会判错。

## 为什么是 O(n)

核心论证一句话：**每个下标作为 `right` 最多进一次，作为 `left` 最多出一次。**

- 外层 `for right` 跑 n 次。
- 内层 `while` 里 `left` 只增不减，一生最多 +n。
- 均摊下来，总操作次数 ≤ 2n 量级 → **时间 `O(n)`**（若字符集很大、用哈希表，还要乘上哈希操作的平均常数）。

对撞指针同理：`left` 只增、`right` 只减，相遇即停，移动总次数 ≤ n。

这就是「看起来像两层循环，复杂度仍是线性」的原因——内层不是「对每个 right 都从 0 重扫」，而是 **接力推进**。

空间：计数数组/哈希通常 `O(字符集)` 或 `O(窗口内不同元素数)`；对撞两数之和额外 `O(1)`。

## 边界与易错

| 边界                       | 处理                                                                |
| -------------------------- | ------------------------------------------------------------------- |
| 空数组 / 空串              | 直接返回 0 或 `[]`，别进循环下标越界                                |
| 全相同字符                 | 最长无重复 → 答案 1；窗口会持续 `left++`                            |
| 窗口长度 0                 | `left == right+1` 时合法空窗，更新答案前确认题意是否允许            |
| 最优窗在开头/结尾          | 检查首轮与末轮是否更新了 `ans`                                      |
| 含负数的「最小长度子数组」 | 若用和做条件且可负，单纯滑动窗口单调性可能失效，要换前缀和/其他结构 |

其它坑：

1. **先排序再对撞** 时，记得总时间至少 `O(n log n)`，且下标映射可能被打乱（要原下标就先记 pair）。
2. **最长/最短** 更新时机写反，样例碰巧过、边界挂。
3. 链表快慢：判断写 `fast != null && fast.next != null`，顺序别反。
4. 窗口题移出元素时漏更新计数器，等于状态和指针脱节。

## 动手前的口述清单

1. 题型：对撞 / 同向 / 窗口？有序还是要先排序？
2. 指针含义：`left/right` 各代表什么边界。
3. 窗口状态：和、计数、valid 种类……
4. 扩缩条件：何时 `right++`，何时 `while` 缩 `left`。
5. 答案时机：最长先修合法再记；最短满足即记再继续缩。

## 小结

1. 双指针是移动策略，滑动窗口是「连续区间 + 状态」的特例；先定性再写码。
2. 对撞依赖有序/单调排除；同向靠职责分离；窗口靠右扩左缩与计数同步。
3. 两数/三数之和：对撞 + 去重；无序先排序则带上 `O(n log n)`。
4. 线性复杂度成立的前提是指针单调、每个元素进出各一次。
5. 边界先想空串、全相同、更新时机；链表快慢细节另见链表篇。
