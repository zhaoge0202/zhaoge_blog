# 链表和树的高频套路有哪些？

> 链表题靠指针骨架，树题靠递归语义；两者都高度模板化，熟练几套骨架比盲目刷题更管用。

## 链表：先搭骨架，再套题

链表没有下标随机访问，操作都落在「改指向」。高频题几乎都能拆成下面几块积木。

### 哑节点（dummy）

当头节点可能被删、被换、被合并掉时，在真正的头前面挂一个假节点：

```text
dummy -> head -> ...
prev = dummy
# 操作时动 prev.next，最后 return dummy.next
```

好处是统一「删头 / 不删头」的分支，少写一堆 `if head is null` 特判。合并有序链表、删除指定值、两两交换节点，几乎都会用到。

### 快慢指针

两个指针步长不同，常用来：

| 目标        | 做法                                            |
| ----------- | ----------------------------------------------- |
| 找中点      | 快两步、慢一步，快到尾时慢在中点                |
| 判环        | 快慢都从 head 走，相遇则有环                    |
| 环入口      | 相遇后一指针回 head，同步各走一步，再相遇即入口 |
| 倒数第 k 个 | 快先走 k 步，再一起走，快到 null 时慢即目标     |

中点题注意「偶数个节点要左中还是右中」，取决于快指针的起点和循环条件，写前用 2、3 个节点手推一遍。

### 反转：三指针迭代

```text
prev = null
curr = head
while curr != null:
  nxt = curr.next
  curr.next = prev
  prev = curr
  curr = nxt
return prev   # 新头
```

画图看每一轮：`curr` 的 next 先存好，再扳向 `prev`，三人一起前移。递归反转也能写，但迭代边界更直观，面试优先迭代。

### 合并有序链表

哑节点 + 双指针比较：

```text
dummy = ListNode(0)
tail = dummy
while l1 and l2:
  if l1.val <= l2.val:
    tail.next = l1; l1 = l1.next
  else:
    tail.next = l2; l2 = l2.next
  tail = tail.next
tail.next = l1 if l1 else l2
return dummy.next
```

归并 K 个有序链表就是在这个基础上堆或分治。

## 代表题 1：反转链表 / 检测环入口

### 反转整条链表

直接套三指针。边界：

- `head == null` 或只有一个节点 → 原样返回；
- 注意返回的是 `prev` 不是 `head`（`head` 已变成尾，next 为 null）。

局部反转（反转 `[m,n]`）先走到 m 前驱，再对区间做同一套三指针，最后把前驱和后继接回去——仍是「反转 + 哑节点」组合。

### 环形链表 II：找入口

1. 快慢指针判环，无环返回 null；
2. 相遇后，一个指针回到 head，另一个停在相遇点；
3. 两指针每次各走一步，再次相遇的节点就是环入口。

正确性直觉：设头到入口距离 `a`，入口到相遇点 `b`，环长 `c`。快指针多走的路程是环长整数倍，可推得「从 head 和从相遇点同步走，会在入口碰面」。面试能讲清 `a` 与「环上剩余」的关系即可，不必死背公式。

**复杂度**：时间 `O(n)`，空间 `O(1)`（不用哈希表存访问记录）。

**边界**：无环、环在头（自环）、环很靠后、单节点无环 / 自环。快指针判空要写 `fast and fast.next`，避免空指针。

## 树：递归返回信息，而不是只写遍历

树题表面花样多，内核是：**每个节点向上返回什么、向下传递什么**。

### 递归三问

对任意树递归，先回答：

1. 函数语义是什么？（返回高度？是否平衡？子树是否包含 p/q？）
2. 空节点返回什么？
3. 左右子树结果如何合并成当前结果？

前序：根 → 左 → 右，适合「带着路径信息往下走」。  
中序：左 → 根 → 右，BST 上是有序序列。  
后序：左 → 右 → 根，适合「先收齐子树信息再算自己」（高度、直径、平衡、LCA）。

层序用队列：

```text
q = deque([root])
while q:
  size = len(q)
  for _ in 1..size:   # 一层
    node = q.popleft()
    # 处理 node
    if node.left:  q.append(node.left)
    if node.right: q.append(node.right)
```

锯齿层序、每层最右节点、层平均值，都只是在「按层」骨架上改处理逻辑。

## 代表题 2：最近公共祖先（LCA）

在二叉树中找节点 `p`、`q` 的最近公共祖先（二者都存在于树中）。

后序思路：自底向上，看左右子树是否分别「找到了目标」。

```text
function lca(root, p, q):
  if root is null or root == p or root == q:
    return root
  left  = lca(root.left,  p, q)
  right = lca(root.right, p, q)
  if left and right:
    return root          # p、q 分居两侧，当前就是 LCA
  return left if left else right
```

语义：

- 返回值 =「在这棵子树里，找到的 p/q/LCA」；
- 左右都非空 → 当前节点是分叉点，即 LCA；
- 只有一侧非空 → 答案在那一侧（另一侧没有目标）；
- 都空 → 这棵子树没有 p/q。

BST 上还能用值域剪枝：都比 root 小走左，都大走右，否则 root 就是 LCA，时间仍是树高量级但常数更小。

**复杂度**：时间 `O(n)`（最坏每个节点访问一次），空间 `O(h)`（递归栈，h 为高度）。

**边界**：

- `p` 或 `q` 就是根 → 根即答案；
- 一个节点是另一个的祖先 → 返回祖先本身（代码里 `root == p or root == q` 已覆盖）；
- 树退化成只有左链或右链 → 仍正确，只是栈深变 `O(n)`；
- 题目若保证节点存在，可不必额外计数；不保证时要改成「返回找到几个目标」的版本。

同类可迁移：**直径**用后序高度更新 `leftH+rightH`；**平衡**高差 > 1 传哨兵；**路径和**前序累加、叶子判断、回溯弹出。

## 复杂度一览

| 问题                | 时间     | 额外空间             |
| ------------------- | -------- | -------------------- |
| 反转链表            | `O(n)`   | `O(1)`               |
| 快慢判环 / 环入口   | `O(n)`   | `O(1)`               |
| 合并两条有序链表    | `O(n+m)` | `O(1)`               |
| 树遍历 / LCA / 直径 | `O(n)`   | `O(h)` 递归或队列    |
| 层序遍历            | `O(n)`   | `O(w)`，w 为最大层宽 |

深树（链状高度 1e5）递归可能栈溢出，考虑迭代或显式栈。

## 边界与画图习惯

链表翻车点：空头、单节点、两节点；快慢混用时的空指针；反转丢新头、局部反转没接前后驱。

树翻车点：空树、单节点、只有左/右链；漏空节点返回值；全局答案误当局部高度、忘了额外变量。

动手习惯：指针题画 3～5 个节点标 `prev/curr/nxt` 或 `slow/fast`；树题先注释返回语义，再写 `left/right` 合并。骨架对了，变种只改合并条件。

## 小结

1. 链表积木：哑节点、快慢指针、三指针反转、有序合并；多数题是组合拳。
2. 环入口 = 快慢相遇 + 一指针回 head 同步走；反转优先迭代三指针。
3. 树题先定递归语义与空节点返回值，后序适合收子树信息（LCA、高度、直径、平衡）。
4. 层序用队列按层处理；BST 可额外利用有序性剪枝。
5. 空头/单节点/链状树/环要手推；画图看指针与返回值，比直接码更少错。
