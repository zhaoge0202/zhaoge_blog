# 动态规划有哪些常见模型？

> DP 不是公式库，而是「最优子结构 + 重叠子问题」的系统化枚举：把大问题拆成可复用的子问题，用状态表消掉重复计算。

## 先抓住本质，再谈模板

很多题第一眼像递归，暴力写完会超时。DP 能加速，前提是两个性质同时成立：

1. **最优子结构**：大问题的最优解，能由子问题的最优解拼出来。
2. **重叠子问题**：递归树里同一子问题被反复算到。

没有重叠就只是分治；没有最优子结构就不能贪心式地合并子结果。常见四件套：

| 要素     | 要回答的问题                 |
| -------- | ---------------------------- |
| 状态     | 用哪些变量描述「子问题」？   |
| 转移     | 当前状态由哪些已算状态推出？ |
| 边界     | 最小规模下答案是多少？       |
| 遍历顺序 | 用到的子状态是否已经就绪？   |

从暴力到 DP 通常三步：先写递归定义 → 加记忆化（自顶向下）→ 改成递推表（自底向上）。记忆化帮你确认状态和转移对不对；递推则更利于分析复杂度和做空间压缩。

## 通用五步模板

写任何 DP 前，按这个顺序过一遍，比一上来开数组更稳：

1. **定义状态**：`dp[...]` 的语义必须一句话说清，例如「前 i 个物品、容量 j 时的最大价值」。
2. **写转移**：枚举「最后一步做了什么」——选或不选、从左来还是从上来、删/插/替换。
3. **定边界**：空串、容量 0、第一个元素、全 0 行等。
4. **定遍历顺序**：保证转移依赖的状态先算完；0-1 背包一维化时容量要倒序。
5. **取答案并考虑压缩**：答案在 `dp[n][m]` 还是 `max(dp[n])`；能否用滚动数组把一维压掉。

状态比代码重要。状态定错，后面怎么优化都救不回来。

## 三类常见模型

面试里绝大多数线性 DP 都能归到下面三类，先认模型再写方程。

### 1. 线性 / 路径型

状态沿着「位置」推进：走到第 i 格、第 i 层、网格 `(i,j)` 的最优。

典型：爬楼梯、打家劫舍、网格最小路径和、三角形最短路径。

共性：

- 状态通常是 `dp[i]` 或 `dp[i][j]`；
- 转移看「能从哪些前驱到达」；
- 边界是起点本身的代价或方案数。

### 2. 背包型

有容量约束，每个物品有体积/代价和价值，问最大价值、能否恰好填满、方案数等。

| 类型     | 每个物品 | 一维转移时容量方向      | 典型题                     |
| -------- | -------- | ----------------------- | -------------------------- |
| 0-1 背包 | 最多一次 | 倒序                    | 分割等和子集、最后一块石头 |
| 完全背包 | 无限次   | 正序                    | 零钱兑换、组合总和 IV      |
| 多重背包 | 有限次   | 可拆成 0-1 或二进制优化 | 有限库存选购               |

识别口诀：物品「用过就不能再用」→ 0-1；「同种可重复」→ 完全。

### 3. 子序列 / 区间型

看两段序列关系，或一段序列内部区间。

- **子序列**：LCS、LIS、编辑距离——状态常带两个下标 `(i,j)`。
- **区间**：回文子串/子序列、戳气球——`dp[l][r]` 表示区间 `[l,r]`，按区间长度从小到大枚举。

这类题状态维度更高，转移也更绕，但骨架仍是五步模板。

## 代表题 1：0-1 背包

有 `n` 件物品，第 `i` 件体积 `w[i]`、价值 `v[i]`，背包容量 `W`，每件最多选一次，求最大价值。

状态：`dp[i][j]` = 只考虑前 `i` 件物品、容量为 `j` 时的最大价值。

转移：

```text
# 不选第 i 件
dp[i][j] = dp[i-1][j]
# 选第 i 件（j >= w[i]）
dp[i][j] = max(dp[i][j], dp[i-1][j - w[i]] + v[i])
```

边界：`dp[0][*] = 0`，`dp[*][0] = 0`（体积或价值非负时）。

一维压缩后只需 `dp[j]`，**容量必须从大到小**：

```text
dp = [0] * (W + 1)
for i in 0..n-1:
  for j = W down to w[i]:
    dp[j] = max(dp[j], dp[j - w[i]] + v[i])
return dp[W]
```

倒序保证用到的 `dp[j - w[i]]` 仍是「上一轮、未选当前物品」的值；若正序，会在同一轮里重复使用当前物品，退化成完全背包。

**复杂度**：时间 `O(nW)`，空间压缩后 `O(W)`。

**边界**：

- `W = 0` 或没有物品 → 答案 0；
- 某件 `w[i] > W` → 永远选不了，转移里自然跳过；
- 若问「能否恰好装满」，把 max 改成布尔或方案数，初始化 `dp[0] = true`。

## 代表题 2：最长递增子序列（LIS）

给定数组 `nums`，求严格递增子序列的最长长度。

朴素 DP：

- 状态：`dp[i]` = 以 `nums[i]` 结尾的 LIS 长度；
- 转移：`dp[i] = max(dp[j]) + 1`，其中 `j < i` 且 `nums[j] < nums[i]`；没有这样的 `j` 则为 1；
- 答案：`max(dp[i])`。

```text
dp = [1] * n
ans = 1
for i in 1..n-1:
  for j in 0..i-1:
    if nums[j] < nums[i]:
      dp[i] = max(dp[i], dp[j] + 1)
  ans = max(ans, dp[i])
return ans
```

时间 `O(n^2)`，空间 `O(n)`。`n` 到 1e5 时需要 `O(n log n)` 的「维护递增尾巴」做法：用数组 `tails` 存当前各长度 LIS 的最小末尾值，对每个数二分找到第一个 `>=` 它的位置替换。面试先把 `O(n^2)` 讲清楚，再提优化即可。

**边界**：

- 空数组 → 0；单元素 → 1；
- 全递减 → 答案 1；全递增 → 答案 n；
- 严格递增还是非严格，比较符要和题意一致。

同类的编辑距离、完全背包零钱兑换，状态从「一维位置」换成「二维两串」或「金额」，五步不变。

## 复杂度与空间压缩

| 模型           | 常见时间         | 常见空间                | 压缩思路                    |
| -------------- | ---------------- | ----------------------- | --------------------------- |
| 线性路径       | `O(n)` / `O(nm)` | `O(n)` / `O(nm)`        | 只保留上一行/前两项         |
| 0-1 / 完全背包 | `O(nW)`          | `O(nW)` → `O(W)`        | 一维 `dp[j]` + 正确遍历方向 |
| LCS / 编辑距离 | `O(nm)`          | `O(nm)` → `O(min(n,m))` | 滚动两行                    |
| LIS 朴素       | `O(n^2)`         | `O(n)`                  | 另有 `O(n log n)` 写法      |

压缩前务必确认「转移只依赖有限个历史层」。依赖搞不清就先别压，正确性优先。

## 容易踩的坑

1. **初始化漏了**：路径题起点、背包 `dp[0]`、字符串空前缀，漏一个全盘错。
2. **遍历顺序反了**：0-1 背包一维必须倒序；区间 DP 必须先短后长；依赖「左边和上边」的网格只能从上到下、从左到右。
3. **容量 / 下标从 0 开始的语义**：`dp` 长度是 `W+1` 还是 `n`，和「前 i 个」是 1-index 还是 0-index，写前统一。
4. **负数与「不可达」**：最小路径和若有障碍，不可达格子要设成无穷大，不能默认为 0。
5. **求方案数 vs 求最优值**：转移里是 `+` 还是 `max/min`，初始化也不同（方案数常 `dp[0]=1`）。

## 不会建模时怎么办？

回到枚举：

1. 暴力递归：函数参数就是候选状态；
2. 看哪些参数真正影响答案，砍掉无关维度；
3. 给递归加缓存，确认有重叠；
4. 再改成数组递推，核对边界和顺序。

如果子问题不重叠，或最优解不能由子最优拼出，就不是 DP——可能是贪心、搜索或图算法。别硬套。

和贪心的边界：贪心每步局部最优且能证明全局；证不了、有反例，就用 DP 做系统枚举。和回溯的边界：要所有方案细节用回溯；只要最优值/计数，DP 通常更合适。

## 小结

1. DP = 最优子结构 + 重叠子问题；先定义状态语义，再写转移。
2. 标准路径：暴力递归 → 记忆化 → 递推 →（可选）空间压缩。
3. 面试三类主模型：线性路径、背包（0-1/完全）、子序列/区间。
4. 五步模板：状态、转移、边界、遍历顺序、答案与压缩。
5. 初始化、遍历方向、不可达表示是高频翻车点；建模卡壳时回到枚举。
