# 面向对象三大特征、重载重写、接口与抽象类怎么理解？

> 面向对象这一块的面试题，考的从来不是背诵定义，而是你能不能说清「谁在编译期定、谁在运行期定」。这篇就围绕这条主线，把封装继承多态、重载重写、接口与抽象类一次讲透。

## 先分清：对象、引用、`new` 到底谁在堆谁在栈

聊面向对象之前，得先把最基础的三个词摆正，不然后面讲多态会绕晕。

```java
Person p = new Person();
```

这一行其实干了两件事：

- `new Person()` 在**堆**里开辟了一块内存，放的是真正的对象实例（字段值都在这里）。
- `p` 是一个**引用**，它待在**栈**里，值是那块堆内存的地址。

打个比方：对象是气球，引用是拴气球的绳子。一根绳子可以不拴气球（`p = null`），也可以拴一个；反过来，一个气球可以被好几根绳子同时拴住（多个引用指向同一对象）。理解了「引用只是地址」，后面的引用拷贝、多态字段访问就都顺了。

顺带区分两个「相等」：`==` 比的是两根绳子指没指向同一个气球（地址），`equals()`（被重写后）比的是两个气球内容一不一样。这块单独有一篇细讲，这里不展开。

## 封装、继承、多态，一句话本质加一个例子

这三个特征不是并列的三条规则，而是层层递进：封装管好自己，继承复用别人，多态让调用方不用关心具体是谁。

**封装**：把对象的内部状态藏起来，只留几个方法给外面用。本质是「隐藏细节、暴露接口」。

```java
public class Account {
    private long balance;              // 余额藏起来，外面动不了
    public void deposit(long amount) { // 只能通过方法改，方法里能做校验
        if (amount <= 0) throw new IllegalArgumentException("金额非法");
        balance += amount;
    }
    public long getBalance() { return balance; }
}
```

好处很实在：余额不会被外部随便改成负数，校验逻辑也集中在一处。像空调一样，你只用遥控器（方法），看不到也不用管内部电路（字段）。

**继承**：用已有的类做基础，扩展出新类，把公共部分抽上去复用。本质是「is-a，子类是一种父类」。狗是一种动物，`Dog extends Animal`。

要记住三点：子类拥有父类全部属性和方法（私有的也算「拥有」，只是访问不到）；子类能加自己的东西；子类能用自己的方式重写父类方法。Java 是**单继承**，一个类只能有一个直接父类。

**多态**：同一个父类引用，指向不同子类对象，调用同一个方法会跑出不同结果。本质是「一个接口，多种实现」。

```java
Animal a = new Dog();
a.sound();   // 汪汪
a = new Cat();
a.sound();   // 喵喵
```

多态的关键是这句话，一定要记牢：**编译期看引用类型，运行期看实际对象**。编译器在编译时只知道 `a` 声明成了 `Animal`，所以它只允许你调用 `Animal` 里有的方法（这就是为什么「只在子类存在、父类没有」的方法编译不过）；而真正执行哪个版本，要等运行时看 `a` 到底指向 `Dog` 还是 `Cat`，这个过程叫**动态绑定**。

### 成员变量和静态方法不参与多态

这是高频陷阱，很多资料只讲「方法多态」就不往下说了。实际上，**只有实例方法是动态绑定的；字段和静态方法都是按引用类型（编译期）解析的**。看这个例子：

```java
class Parent {
    int x = 10;
    static String who() { return "Parent-static"; }
    String type()       { return "Parent-instance"; }
}
class Child extends Parent {
    int x = 20;
    static String who() { return "Child-static"; }
    String type()       { return "Child-instance"; }
}

Parent p = new Child();
System.out.println(p.x);       // 10          —— 字段看引用类型 Parent
System.out.println(p.who());   // Parent-static —— 静态方法看引用类型
System.out.println(p.type());  // Child-instance —— 实例方法动态绑定
```

原因是字段访问和静态方法调用在编译期就绑定好了，编译器认的是 `p` 的声明类型 `Parent`；只有实例方法调用会走虚方法表在运行期分派。所以工程上不要用子类字段去「覆盖」父类同名字段，看着像重写，其实只是隐藏（hiding），非常容易读错。

## 重载 vs 重写：一张表分清

这俩名字像，考点却完全相反，是必考中的必考。先记一句总纲：**重载是编译期的事，重写是运行期的事**。

- **重载（Overload）**：同一个类里，方法名相同、参数列表不同，是「同名不同参」。编译器在编译时就根据你传的实参类型，挑好调用哪个版本，叫**静态绑定**。
- **重写（Override）**：子类把父类的方法重新实现一遍，方法名和参数列表完全一样，是「同名同参不同体」。调用哪个版本运行时才定，叫**动态绑定**。

| 对比点     | 重载 Overload                                | 重写 Override                                                   |
| ---------- | -------------------------------------------- | --------------------------------------------------------------- |
| 发生位置   | 同一个类中（从父类继承来的方法也能参与重载） | 有继承关系的父子类之间                                          |
| 参数列表   | **必须不同**（类型、个数、顺序至少差一样）   | **必须完全相同**                                                |
| 返回值     | 无关，可随意改                               | void 和基本类型必须相同；引用类型可以是原类型的子类（协变返回） |
| 访问修饰符 | 无关，可随意改                               | 不能比父类更严（子类 ≥ 父类）                                   |
| 抛出异常   | 无关，可随意改                               | 受检异常范围不能比父类更大（子类 ≤ 父类）                       |
| 绑定时机   | 编译期静态绑定                               | 运行期动态绑定                                                  |

有个反复被问的细节：**重载只看方法签名，而方法签名 = 方法名 + 参数列表，跟返回值无关**。所以下面这两个不构成重载，反而直接编译报错——因为签名一模一样，编译器无法区分：

```java
int  foo(int a) { ... }
long foo(int a) { ... }   // 编译错误：仅返回值不同，不是重载
```

重写的「协变返回」也值得记一下：如果父类方法返回 `Hero`，子类重写时可以改成返回 `SuperMan`（`Hero` 的子类），但如果返回的是 `int` 这种基本类型或 `void`，就必须原样不动。

还有两个重写的边界要背：`private`、`final`、`static` 修饰的方法**不能被重写**。`private` 子类根本看不见；`final` 明令禁止；`static` 属于类，子类写一个同名同参的静态方法只是「方法隐藏」，不是重写（回顾上面那个 `who()` 的例子就懂了）。

## 接口 vs 抽象类：共同点、区别，以及 Java 8/9 的新变化

先说共同点，其实就两条：都**不能直接实例化**，都能**包含抽象方法**（没有方法体、留给子类实现）。

区别才是重点，很多老资料停留在「接口只能有抽象方法」的旧认知上，这在 Java 8 之后已经不成立了，答错会显得知识过时：

| 对比点     | 接口 interface                                                        | 抽象类 abstract class                                |
| ---------- | --------------------------------------------------------------------- | ---------------------------------------------------- |
| 设计意图   | 约束**行为/能力**（has-a，「能做什么」）                              | 表达**归属关系**（is-a，「是一种什么」），偏代码复用 |
| 继承/实现  | 一个类可**实现多个**接口，接口间也能多继承                            | 一个类只能**继承一个**抽象类（Java 单继承）          |
| 成员变量   | 只能是 `public static final` 常量，必须有初值                         | 可以是任意修饰符的普通字段，能被子类改               |
| 方法（旧） | Java 8 前只能有 `public abstract` 抽象方法                            | 一直可以有抽象方法 + 有实现的普通方法                |
| 方法（新） | Java 8 起支持 `default` / `static` 方法；Java 9 起支持 `private` 方法 | 无此变化，本来就能写具体方法                         |
| 构造方法   | 没有                                                                  | 有（供子类 `super()` 调用，但不能 `new`）            |

关于接口的新特性，稍微展开一下，这几个是版本红利：

- **`default` 方法（Java 8）**：给接口方法一个默认实现，实现类可以直接用、也可以覆盖。它解决的核心痛点是——往一个已经被无数类实现的接口里加新方法，老实现类不用改就能编译通过（向后兼容）。`Collection.stream()` 就是这么加进去的。
- **`static` 方法（Java 8）**：属于接口本身，只能用 `接口名.方法()` 调，实现类不能覆盖它。适合放和接口相关的工具方法。
- **`private` 方法（Java 9）**：给 `default`/`static` 方法抽公共逻辑用，不对外暴露。

至于怎么选：需要**多继承能力**、或者只想约定「你得能做 X」，用接口；有一堆**共享的字段和实现**想让子类复用、且它们确实是「同一族」，用抽象类。一句话——**接口描述「像不像」，抽象类描述「是不是」**。实际工程里两者经常配合：接口定契约，抽象类提供一个骨架实现（比如 `List` 接口 + `AbstractList`）。

## 构造方法有哪些特点

构造方法是专门用来初始化对象的特殊方法，特点可以串成一句话记：

- **名字必须和类名完全一样**。
- **没有返回值，连 `void` 都不能写**——写了 `void` 它就退化成一个普通方法了，这是新手常踩的坑。
- **不能被 `static`、`final`、`abstract` 修饰**：它天生属于实例的初始化过程，加这些修饰符没有意义，编译器也不允许。
- **能被重载，不能被重写**。一个类可以有多个参数不同的构造方法（重载），提供不同的初始化方式；但它跟继承体系无关，谈不上重写。
- **不写就有默认的无参构造**。只要你没声明任何构造方法，编译器会自动补一个无参的；可一旦你写了哪怕一个有参构造，这个默认无参构造就**不再自动生成**了。所以写了有参构造后，如果还需要无参的，记得手动补上，否则 `new Xxx()` 会编译报错。

## 深拷贝、浅拷贝、引用拷贝

这三个词描述「复制一个对象」时的三种程度，用一个带引用字段的例子最好懂。假设有 `Person` 类，里面包了一个 `Address` 对象：

| 类型     | 有没有 new 出新对象 | 内部引用字段怎么处理           | 结果                            |
| -------- | ------------------- | ------------------------------ | ------------------------------- |
| 引用拷贝 | 没有                | ——                             | 两个引用指向**同一个** Person   |
| 浅拷贝   | 有，新 Person       | 只复制地址，共享同一个 Address | Person 是两个，Address 还是一个 |
| 深拷贝   | 有，新 Person       | 连 Address 也复制一份          | Person、Address 全都是两个      |

**引用拷贝**最简单，就是 `Person p2 = p1;`，压根没造新对象，两根绳子拴同一个气球，改一个另一个跟着变。

**浅拷贝**通常靠实现 `Cloneable`、重写 `clone()` 调 `super.clone()` 实现。它会在堆上造一个新 Person，但内部的 `Address` 只复制了引用地址：

```java
Person p1 = new Person(new Address("武汉"));
Person p2 = p1.clone();               // 浅拷贝
System.out.println(p1.getAddress() == p2.getAddress()); // true，共享同一个 Address
```

**深拷贝**要在 `clone()` 里把内部对象也一并克隆：

```java
@Override
public Person clone() {
    Person p = (Person) super.clone();
    p.setAddress(this.getAddress().clone()); // 关键：把 Address 也复制一份
    return p;
}
// 此时 p1.getAddress() == p2.getAddress() 为 false
```

这里要补一个资料常常没说透的边界：上面的深拷贝只做了「一层」。如果 `Address` 内部又包了别的引用对象，你得**递归地**把每一层都克隆，否则深到某一层又变回共享了。正因为 `clone()` 这套机制手写麻烦又容易漏，实际工程里更常用**拷贝构造函数**、或者**序列化再反序列化**（如 JSON 转来转去）来做深拷贝，代码更清晰也不易错。

## 静态方法为什么不能调用非静态成员

答案的钥匙是**时机对不上**：

- 静态成员属于**类**，在类加载阶段就分配好了内存，还没造任何对象就已经存在，直接 `类名.xxx` 就能访问。
- 非静态成员属于**对象实例**，只有等到 `new` 出对象、实例化之后才存在。

所以当静态方法运行时，非静态成员可能**根本还没被创建**。让一个「早就在」的静态方法去访问一个「可能还不存在」的实例成员，这是逻辑上说不通的非法操作，编译器直接拦下。反过来，实例方法能访问静态成员——因为对象都造出来了，属于类的静态成员肯定早就在了，没有时序矛盾。

这也顺带解释了静态方法和实例方法的另一个区别：静态方法用 `类名.方法()` 就能调，不必先 `new` 对象（虽然语法上也允许 `对象.静态方法()`，但容易造成误解，不推荐）；实例方法则必须先有对象才能调。

## 容易踩的坑

- **以为字段也能多态**：`Parent p = new Child()` 时 `p.x` 取的是父类的字段值，静态方法同理。只有实例方法才动态绑定。
- **拿返回值当重载依据**：仅返回值不同、签名相同的两个方法不是重载，是编译错误。
- **接口知识停在 Java 7**：张口就说「接口只能有抽象方法」会暴露知识老化，Java 8 的 `default`/`static`、Java 9 的 `private` 要能说出来。
- **重写和方法隐藏混为一谈**：`static` 方法在子类里同名同参只是隐藏，不是重写，`@Override` 加上去会编译报错。
- **写了有参构造却忘了无参**：默认无参构造只在你「一个构造都没写」时才自动生成，写了有参的就得自己补无参。
- **深拷贝只拷一层**：内部对象嵌套多层时要递归克隆，否则深层仍是共享。
- **`clone()` 没有 `void` 返回值那种误区**：构造方法不写返回类型，别误加 `void`，加了就变普通方法。

## 小结

1. 多态的核心口诀：**编译期看引用类型（能不能调），运行期看实际对象（调哪个）**；字段和静态方法不参与多态，按引用类型解析。
2. 重载是**编译期静态绑定**，同名不同参，只认签名（方法名+参数列表）跟返回值无关；重写是**运行期动态绑定**，遵循「两同两小一大」。
3. 接口表达能力（has-a）、可多实现、Java 8 后有 `default`/`static`、Java 9 后有 `private`；抽象类表达归属（is-a）、单继承、能有普通字段和构造方法。
4. 构造方法名同类名、无返回值、不能被 `static`/`final`/`abstract` 修饰、能重载不能重写；不写则有默认无参构造，写了有参就不再自动补。
5. 引用拷贝共享对象，浅拷贝复制外壳但共享内部引用，深拷贝逐层复制；实际工程优先用拷贝构造或序列化做深拷贝。
