# 面试准备

这里不堆“所有资料”，而是先给复习顺序、回答目标和纠偏方法。

## 建议顺序

1. 先看 [Java 知识库](../java/) 和 [数据库知识库](../database/)，建立主线。
2. 再刷当前四篇核心文章，把答案讲成“原理 + 场景 + 取舍”。
3. 最后看 [博客里的复盘与实战记录](../blog/)，补表达和踩坑经验。

## 复习目标

- 每道题先讲解决什么问题，再讲关键流程和边界。
- 尽量把答案落回真实项目里的配置、风险和排障。
- 对来源有冲突的知识点，优先整理自己的判断而不是照搬。
