# 数据脱敏和字段加密怎么落地？

> 脱敏解决“展示时少暴露”，加密解决“存储和传输时看不懂”。两者目标不同，不能互相替代。

## 哪些数据要保护？

常见敏感数据包括手机号、身份证号、银行卡号、邮箱、地址、访问令牌、密钥和业务隐私字段。

保护方式通常分三层：

| 层次   | 手段       | 例子                   |
| ------ | ---------- | ---------------------- |
| 展示层 | 脱敏       | `138****8888`          |
| 存储层 | 字段加密   | 身份证号密文入库       |
| 使用层 | 权限与审计 | 谁在什么场景查看了明文 |

## 脱敏怎么做？

脱敏要按场景区分：

- 列表页默认脱敏。
- 详情页按权限展示。
- 导出文件更严格，最好审批和水印。
- 日志永远不要打印完整敏感字段。

脱敏规则要稳定，不要每个接口自己写一套字符串截取。

## 字段加密怎么做？

字段加密通常要考虑：

1. 加密算法和模式。
2. 密钥保存位置。
3. 是否需要模糊查询或等值查询。
4. 密钥轮换后历史数据怎么处理。

如果字段需要等值查询，可以额外存一个不可逆摘要列用于匹配，密文字段只用于解密展示。

## 小结

1. 脱敏是展示控制，加密是存储保护。
2. 敏感字段要避免进入日志、缓存、消息和导出文件。
3. 字段加密必须配套密钥管理、轮换和审计。
4. 查询需求要提前设计，不要上线后再硬改密文字段。
