# AI 生成代码后应该怎样验收？

> AI 生成代码不能用“看起来能跑”验收，必须用测试、检查和真实运行证据闭环。

## 最小验收链路

```text
读 diff -> 跑测试 -> 静态检查 -> 关键路径手测 -> 复核日志和边界
```

如果是前端，还要截图或浏览器自动化；如果是后端，要覆盖接口、数据库状态和异常分支。

## 重点看什么？

- 是否改了不该改的文件。
- 是否吞异常或返回假成功。
- 是否漏权限、租户、幂等校验。
- 是否引入 N+1、全表扫描、无界队列。
- 测试是否只覆盖 happy path。

## 验收证据怎么留？

最终回复或 PR 描述里要写清：

- 跑了哪些命令。
- 哪些测试通过。
- 哪些场景手动验证。
- 哪些风险没有覆盖。

这比“已完成”更可信。

## 小结

1. AI 代码必须用证据验收，不能只看输出文本。
2. 测试、静态检查和关键路径运行验证缺一不可。
3. 审查重点是权限、异常、事务、性能和边界条件。
4. 未验证的风险要明确说出来。
